# Workflows: From idea to production

Step-by-step flows so Islamic tech ideas from this board can be taken into production by builders, founders, and teams.

---

## Overview

1. **Pick an idea** from the [Ideas Board](https://ideas.ummah.build) (or suggest one via [GitHub](https://github.com/ummahbuild/ideas)).
2. **Dissect & validate** using templates (problem, solution, users).
3. **Check Islamic authenticity** where relevant (fiqh, sources, transparency).
4. **Research market & competition** (TAM/SAM/SOM, competitors, positioning).
5. **Define brand & messaging** (name, persona, value prop).
6. **Plan go-to-market** (channels, launch, first 90 days).
7. **Build MVP** and iterate with feedback.

Templates live in [../templates/](../templates/). Use them in order or cherry-pick by stage.

## Workflow 1: Idea → MVP (solo or small team)

| Step | Action | Template / doc |
|------|--------|-----------------|
| 1 | Choose idea from board; copy `requirements` into your doc | [ideas.ummah.build](https://ideas.ummah.build) |
| 2 | Dissect: problem, solution, users, MVP scope | [Idea dissection](../templates/idea-dissection.md) |
| 3 | If worship/finance/content: run Islamic authenticity checklist | [Islamic authenticity](../templates/islamic-authenticity.md) |
| 4 | Name and one-liner | [Name/idea generation](../templates/name-idea-generation.md), [Brand](../templates/brand-generation.md) |
| 5 | Define primary persona and messaging | [Target personas](../templates/target-personas.md), [Messaging](../templates/messaging-for-personas.md) |
| 6 | Build MVP (core features only); get 5–10 users | — |
| 7 | Launch and iterate | [Go to market](../templates/go-to-market.md), [Social media](../templates/social-media-marketing.md) |

## Workflow 3: Improve an existing idea on the board

| Step | Action |
|------|--------|
| 1 | Open [Edit existing idea](https://github.com/ummahbuild/ideas/issues/new?template=edit_idea.md) issue |
| 2 | Propose new fields (e.g. `personas`, `relatedIdeas`) or richer `requirements` |
| 3 | Or fork, update `ideas.json` and [schema](../../schema/idea.schema.json), then PR |

See [CONTRIBUTING.md](https://github.com/ummahbuild/ideas/blob/main/CONTRIBUTING.md) and [Comprehensive idea structure](../COMPREHENSIVE_IDEA_STRUCTURE.md). Add `relatedSlugs` to ideas in ideas.json for better Related suggestions on the board.
